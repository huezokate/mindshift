## Vision

Redesign the V100 screen flow to a clean three-step entry sequence:

1. **Welcome** — splash screen (dark, cyberpunk aesthetic regardless of theme)
2. **UI Selection** — dedicated full-screen horizontal carousel where the user swipes through Cyberpunk / Kawaii / Notepad theme cards and taps CHOOSE
3. **Landing** — vent input screen (styled in the chosen theme)

Everything after landing stays the same: lens selection → preview popup → response → share → account.

### What's wrong now

The current build has:
- A theme modal that pops over the lens screen (wrong position in flow)
- The welcome screen exists but goes directly to landing — no dedicated UI selection step
- Theme pill bar was removed but no clean replacement screen exists

### What needs to happen

- Remove the `#overlay-theme` modal and its JS
- Add a new `#screen-ui-select` between welcome and landing
- The UI select screen is a full-screen horizontal snap carousel: one theme card fully visible, adjacent cards peek at ±20px edges
- Swiping (or tapping arrow buttons) moves between Cyberpunk / Kawaii / Notepad cards
- Each card: theme name, 3-line tagline, description paragraph, CHOOSE button
- Tapping CHOOSE applies the theme and navigates to `#screen-landing`
- After CHOOSE the lens screen no longer shows a theme modal — user goes straight to the grid
