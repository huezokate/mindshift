## Context

Research and synthesize the current best practices for mobile-first UX/UI that scales cleanly to web. MindShift is currently capped at 430px — it needs a responsive strategy that preserves the intimate mobile feel at desktop scale.

Key areas to cover:
- Fluid grid systems vs fixed breakpoints for apps like this
- Touch-first interaction patterns that work with mouse/pointer fallback
- Typography scaling (fluid type with clamp() vs step-based)
- Navigation patterns that adapt (bottom nav → sidebar/topbar)
- Canvas/mindmap UX on touch vs mouse (gesture vocabulary)
- Micro-interaction patterns that improve engagement on both platforms
- Progressive disclosure as screen real estate grows

## Acceptance Criteria

- `research.md` covering mobile→web scaling patterns with specific, actionable findings
- Breakpoint strategy recommendation for MindShift's 5-screen wizard + mindmap canvas
- Navigation adaptation plan
- Engagement pattern catalogue relevant to a reflection/journaling app
