## Context

Create `/app/journal` — a protected route that lists the user's saved vent sessions in reverse chronological order. Each session is a card showing the vent text, with all lens responses stacked beneath it as sub-cards. User can expand/collapse a session. Empty state if no entries.

## Acceptance Criteria

- Route at `/app/journal` (server component, auth-gated)
- Fetches `vent_sessions` + related `lens_responses` for the current user from Supabase
- Layout:
  - Page heading: "JOURNAL" (cyberpunk styled)
  - One `SessionCard` per vent session, sorted newest first
  - `SessionCard` shows: date, truncated vent text, expand/collapse toggle
  - When expanded: one `LensResponseCard` per lens response showing figure name + response text
- Empty state: "No entries yet. Vent something and save your first perspective."
- Empty state has a CTA button: "START NOW →" → `/app/onboarding`
- `/app/journal` added to protected routes in Clerk middleware (if not already)
- Link to `/app/journal` added somewhere in the app (response page action row or nav)

## Files

- `src/app/app/journal/page.tsx` — new server component
- `src/components/SessionCard.tsx` — new client component (expand/collapse)
- `src/components/LensResponseCard.tsx` — new client component
- `src/middleware.ts` — ensure `/app/journal` is in the protected matcher
