## Vision

The UI selection screen should feel like stepping into three different worlds — not picking a color scheme. When user swipes between Cyberpunk / Kawaii / Notepad, they see the actual figure portraits from that theme filling the background. The card overlay explains the vibe. The whole thing is infinite-scroll (wraps around).

The welcome screen picks one of the 3 themes at random each session — so the user arrives already immersed in a UI before they even choose. When they enter the selection screen, that theme is already centered.

Once chosen, the theme persists for the entire session. New session = choose again.

### What's wrong now
- `#screen-ui-select` has `background: #080810` hardcoded — Kawaii and Notepad backgrounds look like Cyberpunk
- Welcome screen is hardcoded dark (cyberpunk) regardless of randomly chosen theme
- Carousel is not infinite — hits hard stops at first/last card
- No figure portraits visible in background during selection

### What needs to happen (S-014)
1. Welcome screen: theme-aware (uses CSS vars, styled per randomly chosen theme)
2. UI select background: figure portraits visible behind cards (blurred/faded grid), changing with centered theme
3. Infinite scroll: [cp, kw, np, cp, kw, np, cp, kw, np] — 9 cards (3 repeats), start at center set, silently jump on edge reach
4. Background transitions per centered card theme (portraits + bg color)
