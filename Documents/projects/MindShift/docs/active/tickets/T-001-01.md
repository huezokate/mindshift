## Context

Before any mind map nodes can be rendered, we need a canvas environment that supports pan, zoom, and an SVG layer for drawing connections. This is the foundation everything else builds on.

The current `mindshift.html` renders each "screen" as a `.container` div that fades in/out. The new mind map screen will replace screen 3 (the `.mindmap` section) with a full-viewport canvas.

## Acceptance Criteria

- A full-viewport canvas container renders when the mind map screen is active
- User can pan the canvas by click-dragging
- User can zoom with scroll wheel (or pinch on mobile) — min 0.3x, max 2x
- A `<svg>` overlay layer sits above the canvas for drawing arrows
- The canvas coordinate system (world space vs screen space) is encapsulated in a reusable transform helper so node positioning and arrow drawing can use world coordinates
- Zoom/pan state resets when navigating away from and back to the mind map screen
- Works in latest Chrome, Firefox, Safari (no framework, vanilla JS)
