## Context

All work lands in `V100/V100.html`. The file is a single-file vanilla HTML/CSS/JS prototype (~2700 lines). Three themes (cyberpunk / kawaii / notepad) are controlled by `data-theme` on `#app`. Current screen order is: welcome → landing (wrong — skips UI selection). There is an `#overlay-theme` modal on the lens screen that was a temporary workaround — it must be removed.

## Scope

### 1. Add `#screen-ui-select` (new screen between welcome and landing)

Full-screen horizontal snap carousel. Structure:

```
#screen-ui-select
  .ui-select-track          ← flex row, overflow-x: auto, scroll-snap-type: x mandatory
    .ui-select-card[data-theme-id="cyberpunk"]
    .ui-select-card[data-theme-id="kawaii"]
    .ui-select-card[data-theme-id="notepad"]
  .ui-select-nav             ← arrow buttons + dot indicators
```

Each `.ui-select-card`:
- width: 100% (no peek needed — or calc(100% - 32px) with padding on track for a slight peek)
- scroll-snap-align: center
- Contains: theme name (large, styled per theme), 3-line tagline, description paragraph, CHOOSE button
- The card itself IS styled in that theme's colors (so the cyberpunk card is dark/neon, kawaii is pink/soft, notepad is cream/serif) — this means the card has inline background + color overrides, not relying on `data-theme` on `#app`

Copy per card (from Figma):
- **Cyberpunk**: Name: "CYBERPUNK" | Tagline: "Neon edges. / Dark depths. / Maximum signal." | Desc: "Hack your mind in a high-contrast terminal aesthetic built for focus and intensity. For the ones who think in code and dream in electric light." | Button: "CHOOSE"
- **Kawaii**: Name: "KAWAII" | Tagline: "Soft edges. / Bright energy. / Good vibes only." | Desc: "A playful, candy-colored skin that makes self-growth feel like a treat. For the ones who know joy is the best motivator." | Button: "CHOOSE"
- **Notepad**: Name: "NOTEPAD" | Tagline: "Clean lines. / Clear thinking. / No noise." | Desc: "A distraction-free, paper-light interface that gets out of your way. For the ones who believe clarity is the sharpest tool." | Button: "CHOOSE"

### 2. CHOOSE button behaviour

Tapping CHOOSE on a card:
1. Calls `switchTheme(themeId)` with the card's `data-theme-id`
2. Calls `showScreen('screen-landing')`

### 3. Navigation UI on `#screen-ui-select`

- Left/right arrow buttons (`.ui-select-prev`, `.ui-select-next`) call `scrollUISelect(±1)`
- Three dot indicators (`.ui-select-dot`) — active dot highlights as user scrolls
- Scroll listener (debounced 80ms) detects centered card, updates active dot + previews theme on `#app`

### 4. Remove `#overlay-theme` and all its JS

- Delete the `#overlay-theme` HTML
- Delete CSS for `#overlay-theme`, `.theme-modal-*`
- Delete JS: `openThemeModal()`, `updateThemeModalContent()`, `cycleThemeModal()`, `chooseTheme()`, `themeModalIndex`, `THEME_ORDER`
- Remove the `openThemeModal()` call from `goToLenses()`
- Remove `themeModalName/Tagline/Desc` fields from `THEME_COPY`

### 5. Flow wiring

- Welcome screen: "ENTER THE SPACE" button → `showScreen('screen-ui-select')`
- `goToLanding()` from non-welcome screens goes to `screen-landing` as normal
- On `#screen-ui-select` load: scroll to card matching `currentTheme`, preview that theme
- Add a back button on `#screen-ui-select`: `← Back` → `showScreen('screen-welcome')`

### 6. Card styling (self-contained, not relying on data-theme)

Each `.ui-select-card` must look correct regardless of the current `data-theme` on `#app`. Use inline styles or scoped class overrides per `data-theme-id`:

```css
.ui-select-card[data-theme-id="cyberpunk"] { background: #0D0D1A; border: 1.5px solid rgba(176,76,255,0.6); ... }
.ui-select-card[data-theme-id="kawaii"]    { background: #ffffff; border: 2px solid #111; border-radius: 20px; ... }
.ui-select-card[data-theme-id="notepad"]   { background: #fdfdfb; border: 1.5px solid #c8bfb4; ... }
```

Name, tagline, desc text colors also scoped per `[data-theme-id]`.

## Acceptance Criteria

- Navigating from welcome taps "ENTER THE SPACE" and lands on the UI selection carousel
- All 3 theme cards are visible/swipeable; adjacent cards peek at the edges
- Active dot indicator tracks the centered card
- Scrolling a card into center previews that theme across `#app` (so landing, lens etc already look right when you arrive)
- Tapping CHOOSE applies theme and goes to landing
- `#overlay-theme` no longer exists anywhere in the file
- No JS errors in console
- goToLenses goes straight to grid, no modal
