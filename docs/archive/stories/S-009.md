## Vision

Build all 5 MindShift app screens (Landing, Wizard, Mind Map, Lens Selection, Response) as Figma designs in all 3 UI themes (Cyberpunk, Kawaii, Notepad), with an interactive theme-toggle switcher visible on every screen.

The landing screen (partially built in session) shows how each theme transforms the same UX: same content architecture, radically different visual language. The theme toggle is the hero feature — users pick their aesthetic before they begin the 5-year vision exercise.

### Screens in scope

1. **Screen 1 — Landing**: Hero, tagline, UI theme switcher, portrait teaser, CTA button
2. **Screen 2 — Wizard / Vision Input**: "In 5 years…" prompt, text input, category chips, progress indicator
3. **Screen 3 — Mind Map Canvas**: Spatial canvas, 7 organic-shaped category nodes, center hub, curved connections
4. **Screen 4 — Lens Selection**: Portrait grid (3×3), figure name + era, active selection state
5. **Screen 5 — AI Response**: Response terminal / card, figure portrait, action row (save, share, new lens)

Each screen × 3 themes = 15 Figma frames total (390×844 mobile).

### Theme toggle behavior

The UI switcher component is always visible. Switching themes morphs the entire UI — background, typography, borders, colors, effects — while the content stays identical. This is the core interaction to spec clearly.
