## Context

All work in `V100/V100.html`. Current problems:
- `#screen-ui-select` has `background: #080810` hardcoded → Kawaii/Notepad cards show cyberpunk dark background behind them
- Welcome screen is fully hardcoded dark (non-theme-aware)  
- Carousel stops hard at first/last card
- No portraits visible on selection screen background

## Scope

### 1. Random theme on load

On page load, pick one of 3 themes at random:
```js
const SESSION_THEMES = ['cyberpunk', 'kawaii', 'notepad'];
const sessionTheme = SESSION_THEMES[Math.floor(Math.random() * SESSION_THEMES.length)];
switchTheme(sessionTheme);  // sets data-theme on #app immediately
```
No localStorage — resets each session.

### 3. UI select screen — theme-aware background + portrait grid

The background of `#screen-ui-select` should:
- Fill with the background color of the CURRENTLY CENTERED card's theme (using hardcoded values since the card themes are self-styled)
- Show portrait images of the CURRENTLY CENTERED card's theme blurred/faded in the background

Implementation:

**Background element:**
```html
<div class="ui-select-bg" id="uiSelectBg">
  <!-- JS renders 15 portrait img elements here, blurred and faded -->
</div>
```

**CSS:**
```css
#screen-ui-select { position: relative; overflow: hidden; }
.ui-select-bg {
  position: absolute; inset: 0; z-index: 0;
  display: grid; grid-template-columns: repeat(3, 1fr);
  gap: 8px; padding: 8px;
  opacity: 0.18; filter: blur(3px) saturate(0.8);
  transition: opacity 0.3s;
  pointer-events: none;
}
.ui-select-bg img { width: 100%; aspect-ratio: 1; object-fit: cover; }
.ui-select-bg-overlay {
  position: absolute; inset: 0; z-index: 1;
  /* gradient overlay from bg color to transparent — obscures edges */
  pointer-events: none;
}
/* cyberpunk bg: dark overlay */
.ui-select-bg-cyberpunk { background: #080810; }
/* kawaii: light overlay */
.ui-select-bg-kawaii { background: #edf2ff; }
/* notepad: cream overlay */
.ui-select-bg-notepad { background: #fdfdfb; }
```

The `ui-select-bg` gets filled with portrait images when `initUISelect()` runs. Use the portrait paths for the current centered theme. When centered card changes (via scroll listener), call `updateUISelectBg(themeId)` which:
1. Sets `#screen-ui-select` background color (hardcoded per theme: cyberpunk=#080810, kawaii=#edf2ff, notepad=#fdfdfb)
2. Updates portrait images to that theme's portraits
3. Updates dot indicators

**Portrait rendering:**
```js
function updateUISelectBg(themeId) {
  const bg = document.getElementById('uiSelectBg');
  if (!bg) return;
  bg.innerHTML = '';
  FIGURES.forEach(fig => {
    const img = document.createElement('img');
    img.src = fig.portraits[themeId] || fig.portraits.cyberpunk;
    img.alt = '';
    img.loading = 'lazy';
    bg.appendChild(img);
  });
  // background color of screen
  const bgColors = { cyberpunk: '#080810', kawaii: '#edf2ff', notepad: '#fdfdfb' };
  document.getElementById('screen-ui-select').style.background = bgColors[themeId] || '#080810';
}
```

### 4. Infinite scroll carousel — 9 cards (3×3 repeat)

Replace 3 HTML cards with 9 (three sets of [cyberpunk, kawaii, notepad]):

```html
<!-- Set A (before) -->
<div class="ui-select-card" data-theme-id="cyberpunk">...</div>
<div class="ui-select-card" data-theme-id="kawaii">...</div>
<div class="ui-select-card" data-theme-id="notepad">...</div>
<!-- Set B (center — start here) -->
<div class="ui-select-card" data-theme-id="cyberpunk">...</div>
<div class="ui-select-card" data-theme-id="kawaii">...</div>
<div class="ui-select-card" data-theme-id="notepad">...</div>
<!-- Set C (after) -->
<div class="ui-select-card" data-theme-id="cyberpunk">...</div>
<div class="ui-select-card" data-theme-id="kawaii">...</div>
<div class="ui-select-card" data-theme-id="notepad">...</div>
```

`initUISelect()` scrolls to center set (index 3,4,5) matching current theme. 

Scroll listener detects centered card. When centered card is in Set A (index 0,1,2) or Set C (index 6,7,8), silently jump to the equivalent card in Set B (index 3,4,5) using `scrollLeft` without animation.

The "silent jump" trick:
```js
// After 80ms debounce, if centered card idx < 3 or > 5:
const equivalentIdx = ((centeredIdx % 3) + 3); // maps to set B
const card = cards[equivalentIdx];
track.scrollLeft = card.offsetLeft - (track.offsetWidth - card.offsetWidth) / 2;
```

### 5. Dots — 3 dots only (one per theme, not per card)

Dots show position within a single set (0=cyberpunk, 1=kawaii, 2=notepad).
Active dot = `centeredIdx % 3`.

### 6. scrollUISelect(direction)

Jump to adjacent card in current set using centered card detection. No changes needed if infinite scroll is implemented correctly.

## Acceptance Criteria
- Welcome screen matches randomly chosen theme (kawaii welcome has soft pink/purple styling, notepad has cream/ink, cyberpunk has neon)
- UI select background shows blurred portrait grid matching centered card's theme
- Swiping left/right from notepad wraps to cyberpunk (infinite, no hard stop)
- Dot indicator tracks position mod 3 (not total card index)
- Tapping CHOOSE applies theme, goes to landing, theme persists for session
- New page load = new random theme (no localStorage persistence)
