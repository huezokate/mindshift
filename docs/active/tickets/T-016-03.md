## Context

Add a Save button to `/app/response`. Appears after the typewriter animation completes (same timing as the existing action row). On tap: calls `POST /api/save-response`, stores returned `sessionId` in sessionStorage as `ms_session_id`, shows a saved confirmation state. If the user taps "Try another lens" after saving, the session_id is passed along so the next lens response saves under the same session.

## Acceptance Criteria

- Save button appears in the action row after typing animation completes
- Button label: "SAVE TO JOURNAL" (uppercase, styled with `var(--cyan)` border matching the cyberpunk design system)
- Loading state while POST is in-flight (button disabled, label "SAVING...")
- On success: button changes to "SAVED ✓" (non-interactive) and `ms_session_id` written to sessionStorage
- On error: button resets, shows a brief error message below the action row
- "Try another lens" preserves the session: passes `ms_session_id` context so the lens page and subsequent save can reference it
- Does not block navigation — user can still tap "New session" or "Try another lens" without saving
