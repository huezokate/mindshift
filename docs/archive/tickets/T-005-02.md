## Context

The `ventToFigure` function currently shows a hardcoded placeholder after a 2-second timeout. Wire it to the Claude API (claude-haiku-4-5-20251001) so each figure responds in-character to the user's vent text.

Each figure needs a system prompt that captures their voice and worldview. The API key is entered by the user and stored in localStorage. The response populates the existing deepDiveContent sections and uses the existing reveal animation.

## Acceptance Criteria

- API key input (stored in localStorage, clearable)
- Each of the 9 figures has a distinct system prompt
- `ventToFigure` calls Claude API and renders the response in deepDiveContent
- Loading overlay stays visible during the API call
- Graceful error state if API call fails (shows message, returns to pick page)
- No regression to the non-vent flow (canvas lens, generic personas)
