## Vision

QA stress-test of V100 on real device conditions: small-screen scroll paths, overlay containment, navigation dead-ends, and visual noise reduction. Every broken interaction and unnecessary decoration fixed so the prototype is clean enough for real phone user testing.

### Issues in scope (QA findings)

**P0 — Broken interactions:**
- Response screen: sticky bottom nav clips action row and response card tail — no scroll clearance
- Preview + loading overlays use `position: fixed` and bleed outside 420px shell on desktop
- Lens screen: no back button — navigation dead-end (can't return to edit vent)
- "New" nav button doesn't clear textarea — old vent persists on new session

**P1 — Visual defects:**
- Cyberpunk h1 text gradient — unnecessary, replace with flat `--c1`
- Kawaii `add-card` linear-gradient background — replace with flat `var(--bg-card2)`
- Initial HTML textarea `maxlength="800"` causes flash of wrong counter before script override
- `quote-text` fixed height + internal scroll — replace with line-clamp for cleaner overflow

**P2 — Polish:**
- "Continue" bottom nav uses `alert()` — replace with styled in-screen teaser toast
- `ventError` appears abruptly — add slide-down transition
- Bottom nav "Try another" label truncates on small screens — trim to fit
