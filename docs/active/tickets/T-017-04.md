## Context

Update `/app/response` to align with the Figma card system while keeping the existing typewriter animation and logic.

The current response screen already has the right structure (figure header, terminal card, action row). It just needs the card border/radius/shadow to match the Figma card system, and the figure avatar to show the actual portrait photo.

## What needs to change

1. **Figure avatar** — currently shows the first letter of the figure's name as text. Replace with the portrait photo:
   - 80px circle
   - `var(--fig-avatar-border)` border
   - `var(--fig-avatar-shadow)` shadow
   - `var(--fig-avatar-grad)` background gradient
   - `<img src={getFigureImg(figure, theme)} />` — figure loaded from `ms_figure_id` in sessionStorage matched against `FIGURES`

2. **Terminal card** — already uses `var(--card-*)` tokens, looks correct. No layout changes needed (already fixed the stretch bug).

3. **Action row** — add "SAVE TO JOURNAL" primary button above "Try another lens" (as per S-016 T-016-03). This is the soft auth gate: clicking it prompts sign-in if not logged in, then saves.

## How to get figure image

```tsx
import { useTheme } from '@/lib/theme'
import { FIGURES, getFigureImg } from '@/lib/figures'
const { theme } = useTheme()
const figureId = sessionStorage.getItem('ms_figure_id')
const figure = FIGURES.find(f => f.id === figureId)
// <img src={getFigureImg(figure, theme)} />
```
