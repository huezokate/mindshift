## Context

Set up environment variables and Supabase DB schema for V200. The scaffold at `/V200/` already has `.env.local.example` with all required keys. Need to create `.env.local` from it and run the Supabase migration.

## Acceptance Criteria

- `.env.local` created from `.env.local.example` with real values filled in
- Supabase schema created (SQL below runs without errors):

```sql
-- profiles table (synced from Clerk via webhook or on first sign-in)
create table if not exists profiles (
  clerk_user_id text primary key,
  is_pro boolean default false,
  theme text default 'cyberpunk',
  created_at timestamptz default now()
);

-- vision_entries table
create table if not exists vision_entries (
  id uuid primary key default gen_random_uuid(),
  clerk_user_id text references profiles(clerk_user_id) on delete cascade,
  prompt text not null,
  goals jsonb not null default '{}',
  created_at timestamptz default now()
);

-- responses table (saved AI responses)
create table if not exists responses (
  id uuid primary key default gen_random_uuid(),
  clerk_user_id text references profiles(clerk_user_id) on delete cascade,
  vision_entry_id uuid references vision_entries(id) on delete cascade,
  figure_id text not null,
  response_text text not null,
  created_at timestamptz default now()
);

-- RLS policies
alter table profiles enable row level security;
alter table vision_entries enable row level security;
alter table responses enable row level security;

create policy "Users see own profile" on profiles for all using (clerk_user_id = requesting_user_id());
create policy "Users see own entries" on vision_entries for all using (clerk_user_id = requesting_user_id());
create policy "Users see own responses" on responses for all using (clerk_user_id = requesting_user_id());
```

- `npm run dev` starts without env errors
