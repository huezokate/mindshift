## Context

Every screen in V100 uses the same copy regardless of theme. Cyberpunk should feel terse and terminal ("VENT IT OUT > _"). Kawaii should feel warm and expressive ("share what's on your heart ✨"). Notepad should feel thoughtful and journal-like ("Write it out..."). Loading states also need per-theme personality.

## Acceptance Criteria

- `research.md` lists every string in V100 that should be theme-variant
- `design.md` writes final copy for all three themes across: landing headline, vent placeholder, vent label, "Select the Lens" button, "Pick a Lens" heading, loading subtitle
- Implementation updates `switchTheme()` to also swap copy strings
- Loading overlay gets per-theme animation: cyberpunk = blinking cursor dots, kawaii = bouncing dots with color, notepad = ellipsis fade
- No copy regression — strings must degrade to current defaults if theme unknown
