## Vision

Polish V100 into a high-fidelity interaction prototype that reliably tests two things on a real phone:

1. **Theme switching** — the transition between Cyberpunk / Kawaii / Notepad feels intentional and immersive, not like a CSS class swap. Each theme should have its own animation personality.
2. **Core interaction flow** — vent → lens selection → response feels fluid, with each step having per-theme micro-animations, copy tone, and tactile feedback.

V100 currently has all three design systems working, all 15 figures, mock responses, and a clean screen flow. This story upgrades the *feel* — the transitions, animations, copy, actions, and edge cases that make the prototype feel like a real app rather than a styled HTML file.

### Improvements in scope

- **Theme transition animation** — morphing effect between themes (not instant snap)
- **Per-theme micro-copy** — landing headline, vent placeholder, button labels adapt per theme
- **Typewriter response reveal** — each theme gets its own text animation personality
- **Response action buttons** — Save, Decorate, Socials icons + labels; Continue conversation
- **Account gate logic** — 4th lens attempt triggers account creation screen (per Figma spec)
- **Mobile haptics + scroll** — `navigator.vibrate()` on key taps; scroll momentum in lens grid
- **Loading animation personality** — per-theme loading states (terminal typing / sparkle / handwriting)
- **Lens grid UX** — selected card persists after preview closes; count badge; figure tag legibility
