## Context

Create `POST /api/save-response` — the endpoint the response page calls when the user hits Save. It upserts a `vent_sessions` row (idempotent via session_id from sessionStorage) and inserts a `lens_responses` row for the current lens.

Upsert on `vent_sessions` so that applying a second lens to the same session doesn't create a duplicate vent row.

## Acceptance Criteria

- `POST /api/save-response` requires Clerk auth; returns 401 if unauthenticated
- Body: `{ sessionId?, ventText, figureId, responseText, theme }`
  - If `sessionId` is provided and exists → insert `lens_responses` only
  - If no `sessionId` → insert `vent_sessions` first, then `lens_responses`, return `{ sessionId }`
- Returns `{ sessionId, responseId }` on success
- Uses `getSupabaseAdmin()` (service role, bypasses RLS for server-side writes)
- Duplicate lens on same session is prevented (unique constraint on `session_id + figure_id`)
