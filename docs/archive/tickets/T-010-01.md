## Context

Audit V100/V100.html as it stands. Map every screen, interaction, animation, and edge case. Identify what works, what's missing, what feels wrong on mobile. This is the foundation for all five implementation tickets.

## Acceptance Criteria

- `research.md` enumerates all current screens, interactions, and JS functions
- Documents what each theme currently does vs. what it should feel like
- Lists every missing interaction (animations, gates, edge cases) with severity
- Notes mobile-specific issues (tap targets, scroll, haptics, overflow)
- References Figma sticky notes (loading time warning, account gate triggers)
