## Context

Once all canvas components exist, this ticket wires the new mind map canvas into the full app flow, replacing the old linear mind map screen (page 3 in the current HTML). The existing onboarding (pages 1-2) and the persona lens screens (pages 4-5) are preserved and connected to the new canvas.

This ticket also adds the "Choose Your Lens →" button to the canvas that navigates to the persona selection screen, and the "← Back to Map" button that returns to the canvas.

## Acceptance Criteria

- Completing the onboarding form (page 2) navigates to the new canvas (replacing old page 3)
- User's onboarding answers are passed into the canvas and populate: central hub text, career node content (from q1 and q4 selection), and remaining nodes with sensible defaults derived from the answers
- Canvas has a "Choose Your Lens →" button that navigates to the persona screen (page 4)
- Persona screen's "← Back to Map" returns to the canvas with state preserved (nodes, images, edits intact)
- The old linear mind map page 3 HTML/JS is removed or gated behind a flag
- Back navigation from canvas to onboarding (page 2) works and warns user that returning will reset the map
- The new `mindshift.html` (or a new `mindshift-v2.html`) is a single deliverable file that runs by opening in a browser with no server
