## Context

Build the onboarding wizard at `/app/onboarding`. This is the 5-step vision input flow — the core UX that feeds the mind map. Must match Figma cyberpunk design.

## Flow Steps

1. **Vent** — large text input: "What's on your mind?" (280 char limit, cycling placeholder examples)
2. **Career goals** — "In 5 years, my career looks like..." (bullet list input, max 3)
3. **Life goals** — Same pattern for creativity, health, relationships, travel, finances, living (one category per step or all on one screen TBD from Figma)
4. **Figure pick** — 9-figure grid (portraits), tap to select one
5. **Confirm** — summary card, "MAP MY MIND" CTA → saves to Supabase, redirects to `/app/mindmap`

## Acceptance Criteria

- Wizard uses URL state or React state for step tracking
- Step progress indicator (dots or line) at top
- Each step has smooth Framer Motion slide transition
- Form validates before advancing (min chars, required field)
- On step 5 submit: POST to `/api/generate-response` with vent text + selected figure
- Response + vision data saved to Supabase (`vision_entries` table)
- Redirect to `/app/mindmap` with entry ID in query param or session storage

## Components

- `src/components/wizard/WizardShell.tsx` — step container + progress bar
- `src/components/wizard/VentInput.tsx` — step 1 large textarea
- `src/components/wizard/GoalInput.tsx` — bullet list input for each category
- `src/components/wizard/FigureGrid.tsx` — 3×3 portrait grid (reuse existing figure data)
- `src/components/wizard/ConfirmStep.tsx` — summary + submit
