## Context

Design the layout architecture for all 5 screens, creating reusable layout zones that work across all 3 themes. Define: what components are shared across themes (structure), what components are theme-specific (style), and how the theme toggle is wired. Produces the blueprint Figma screen builders will follow.

## Acceptance Criteria

- `design.md` defines layout zones for each of the 5 screens
- Specifies which elements are theme-invariant (layout, content) vs. theme-variant (style, color, font, effects)
- Defines the theme-toggle component placement and interaction pattern
- Defines Figma frame naming conventions and canvas positioning for all 15 screens
- Specifies the portrait asset strategy: which figures appear on landing vs. lens grid
