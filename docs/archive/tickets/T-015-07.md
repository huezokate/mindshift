## Context

Wire up Stripe checkout for the "Go Pro" upgrade. Free users get 3 lens responses; pro users get unlimited. Stripe webhook updates Supabase `profiles.is_pro`.

## Acceptance Criteria

- `POST /api/create-checkout` creates a Stripe Checkout session with `clerk_user_id` in metadata
- Stripe webhook at `POST /api/webhook/stripe` handles `checkout.session.completed` → sets `is_pro = true`
- Handles `customer.subscription.deleted` → sets `is_pro = false`
- Usage counter tracked in Supabase or Clerk metadata (increment on each response saved)
- After 3 free uses, response page shows upgrade modal (cyberpunk styled)
- Upgrade modal: price, feature list, "UNLOCK NOW" → Stripe Checkout redirect
- Success redirect: `/app/response?upgraded=true` shows "PRO UNLOCKED" flash

## Files

- `src/app/api/create-checkout/route.ts` — new API route
- `src/app/api/webhook/stripe/route.ts` — already scaffolded, needs usage logic
- `src/components/ui/UpgradeModal.tsx` — new component
