## Context

The lens selection grid is the most interaction-dense screen. Current state: 15 cards in a 3-column grid, tap opens preview popup, confirm selects. Issues: selected card loses its highlight after popup closes, figure tag text is very small (9px), no visual count of how many lenses used, no empty state if portraits fail to load, Che Guevara preview card has "hover / tap" suffix in its name (debug artifact).

## Acceptance Criteria

- `research.md` documents current grid card states, tap flow, and visual hierarchy
- `design.md` specifies: selected state that persists post-popup, count indicator position, tag legibility fix, portrait fallback (initials circle)
- Implementation: selected figure card stays highlighted after preview popup closes
- Implementation: figure tag font bumped to 10px minimum across all themes
- Implementation: portrait `<img>` has `onerror` fallback showing figure initials in a styled circle
- Implementation: lens use count badge near "Pick a Lens" heading ("2 of 3 free")
- Implementation: figure name "Che Guevara" card has no suffix (debug text cleaned)
