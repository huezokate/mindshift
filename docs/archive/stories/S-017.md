## Vision

Rebuild all app screens to match the Figma designs pixel-perfectly. Figma source: https://www.figma.com/design/Mubv0Ghdm2SPxF42JVsX8M/MindShift?node-id=470-2725

The Figma shows Kawaii-themed screens. Since the app is theme-aware (CSS custom properties via `data-theme`), components must use `var(--)` tokens — NOT hardcoded hex values — so they automatically adapt across all three themes (kawaii, cyberpunk, notepad).

**Infrastructure already done (do not redo):**
- Portrait images copied to `public/portraits/kawaii/` and `public/portraits/cyberpunk/` (15 each)
- `figures.ts` updated: `Figure` type now has `imgKawaii` + `imgCyberpunk` fields + `getFigureImg(figure, theme)` helper
- `tokens-kawaii.css` expanded with `--fig-avatar-border`, `--fig-avatar-shadow`, `--fig-avatar-grad`, `--input-header-bg`, `--btn-secondary-*`

**Design token reference (Kawaii, from Figma):**
| Token | Value |
|---|---|

| `--card-bg` | `#ffffff` |

| `--card-bl/br` | `2px/4px solid #400b14` |
| `--card-shadow` | `inset 4px 0 0 0 rgba(64,11,20,0.5)` |
| `--btn-bg` | `#ffe2ac` |
| `--btn-secondary-bg` | `#e5fcfa` |
| `--input-header-bg` | `#e5fcfa` |
| `--input-shadow` | `inset 4px 0 0 0 #49dbc8` |
| `--fig-border` | `2px solid #400b14` |
| `--fig-radius` | `32px` |
| `--fig-avatar-border` | `2px solid #ff50c5` |
| `--fig-avatar-grad` | `linear-gradient(135deg, #c8dcf9, #d9c8f9)` |
| `--fig-name-unsel` | `#7e2091` |
| `--fig-desc` | `#9490b8` |

**Typography (Kawaii):**
- `--font-display` = Nunito Sans — headings (Bold 28/18/12px)
- `--font-body` = Nunito Sans — body (Regular 14px, 10px)
- `--font-btn` = Fredoka Medium 14px, tracking 0.2px

**How to get figure image in a component:**
```tsx
import { useTheme } from '@/lib/theme'
import { getFigureImg } from '@/lib/figures'
const { theme } = useTheme()
<img src={getFigureImg(figure, theme)} alt={figure.name} />
```
