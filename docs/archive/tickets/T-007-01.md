## Context

MindShift's current copy is functional but rough. The Figma "Super final UI for cyberpunk" screens (node 293-1473) have placeholder text, weak CTAs, and generic microcopy. This ticket researches UX writing best practices for emotional/reflective apps and produces an improved copy spec for all 6 screens.

Key areas:
- CTA language: first-person ownership, action clarity, emotional pull
- Placeholder copy: conversational, non-intimidating, curiosity-forward
- Onboarding/account creation: value framing over feature listing
- Microcopy: character counters, "or" dividers, navigation labels
- Tone: cyberpunk-mystical but human, not cold or clinical

## Acceptance Criteria

- `research.md`: UX copy principles for reflection/journaling apps, examples from comparable products
- `design.md`: copy decisions for every text node across all 6 screens with before/after
- `structure.md`: final copy spec organized by screen, ready to implement
- `plan.md`: implementation sequence for Figma push
