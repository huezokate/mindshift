## Vision

Establish a proper design system for MindShift's canvas mind map — defined as Figma components and implemented as CSS design tokens in code.

The old UI (the deep purple/lavender presentation screens) has strong visual identity: rich dark backgrounds, periwinkle accents, warm pastels used sparingly. The canvas currently treats node colors and typography as one-off hardcoded values. This story replaces all of that with a token-based system that is consistent, reviewable, and extensible.

The big fix embedded here: complex clip-path shapes (triangle, diamond, blob, pentagon) currently clip node text content — a structural bug. The design system solves this by separating the shape layer (::before pseudo-element) from the text layer, so content is never hidden.
