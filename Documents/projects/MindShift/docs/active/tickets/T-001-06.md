## Context

In the Figma lo-fi, several nodes contain mood board images — photos the user has added to visualize their goals (e.g., Scandinavian architecture photos in the "living situation" node, travel photos, car/lifestyle images in finances). This turns the mind map into a personal vision board.

Each node should have an "add image" affordance that lets the user upload images from their device. Images are stored as base64 data URLs in session state and displayed as a collage grid inside the node.

## Acceptance Criteria

- Each expanded node shows an "Add image +" button
- Clicking it triggers a file input (image files only)
- Selected images appear as a small collage grid inside the node (max 4-6 visible, rest hidden)
- Images persist in memory for the session
- Images can be removed (click × on image)
- The collapsed node shows a small image count badge if images have been added
- The living situation / travel nodes have this affordance prominently (they show the most images in the lo-fi)
- No server upload — client-side only, base64
