## Context

Theme switching in V100 is instant — data-theme flips, CSS variables update, done. For a prototype testing theme UX on phone, this needs to feel immersive. Each theme should have its own transition personality that previews the aesthetic before it lands.

## Acceptance Criteria

- `research.md` documents current switching mechanism and performance constraints
- `design.md` proposes 3 transition approaches with tradeoffs (flash, slide, morph)
- `plan.md` specifies exact animation: timing, easing, what fades vs. what transforms
- Implementation adds transition to `switchTheme()` without blocking interaction
- On low-power devices / `prefers-reduced-motion`, degrades gracefully to instant swap
